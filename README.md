# img2pdf

固定 A4 四宫格图片转 PDF 批处理工具。

这个项目不是通用图片转 PDF 工具。它只服务一个固定工作流：用户传入一个根目录，根目录下面是一层分组目录，每个分组目录里直接放图片。每个分组目录按每 4 张图片生成一个 PDF，每个 PDF 只有一页 A4，页面固定为 2x2 四宫格。

## 行为规则

- 支持图片格式：`jpg`、`jpeg`、`png`。
- 只扫描输入根目录下的直接子目录。
- 根目录里的图片不处理，更深层子目录里的图片也不处理。
- 每个直接子目录按每 4 张图片生成一个或多个 PDF。
- PDF 文件名使用图片所在目录的最后一级名称；第 2 个 PDF 在目录名后追加 `2`，第 3 个追加 `3`，依此类推。
- 如果多个目录名清理后冲突，后续文件自动追加 `_2`、`_3`。
- 每个目录里的图片按路径排序，然后按每 4 张分组。
- 不足 4 张时仍使用 2x2 四宫格；哪怕只有 1 张图，其他 3 个槽位也不绘制任何内容。
- 页面固定为 A4 竖版，不提供其他页面尺寸或横版模式。

## 使用

```bash
cargo run --release -- <输入目录> <输出目录>
```

参数都可以省略：

- 不传输入目录时，默认扫描当前目录。
- 不传输出目录时，默认输出到 `备份/YYYYMMDD_pdfs`。

示例：

```bash
cargo run --release -- ./images ./pdfs
```

目录示例：

```text
images/
├── group-a/
│   ├── 01.jpg
│   ├── 02.jpg
│   ├── 03.jpg
│   ├── 04.jpg
│   ├── 05.jpg
│   ├── 06.jpg
│   ├── 07.jpg
│   └── 08.jpg
└── group-b/
    ├── 01.png
    └── 02.png
```

输出结果：

```text
pdfs/
├── group-a.pdf   # 包含 01-04
├── group-a2.pdf  # 包含 05-08
└── group-b.pdf  # 右下两个槽位为空
```
